linux_privesc_decision_tree:
  entry_condition: Low-privilege shell obtained on Linux host
  steps:
  - id: LINUX-STEP-01
    question: Can the current user run commands via sudo?
    commands:
    - sudo -l
    if_yes: LINUX-STEP-01A
    if_no: LINUX-STEP-02
  - id: LINUX-STEP-01A
    question: Do allowed binaries or scripts map to known escape patterns or weak
      script execution?
    look_for:
    - NOPASSWD
    - interpreters
    - archive tools
    - editors
    - custom scripts
    next: LINUX-ROOT-CANDIDATE
  - id: LINUX-STEP-02
    question: Are there unusual or abusable SUID binaries?
    commands:
    - find / -perm -4000 2>/dev/null
    if_yes: LINUX-STEP-02A
    if_no: LINUX-STEP-03
  - id: LINUX-STEP-02A
    question: Do any SUID binaries map to known local privilege escalation behavior
      or custom code abuse?
    look_for:
    - unexpected interpreters
    - custom binaries
    - GTFOBins-aligned tools
    next: LINUX-ROOT-CANDIDATE
  - id: LINUX-STEP-03
    question: Are root-owned cron jobs or scheduled scripts writable or predictable?
    commands:
    - cat /etc/crontab
    - ls -la /etc/cron.*
    - pspy64
    if_yes: LINUX-STEP-03A
    if_no: LINUX-STEP-04
  - id: LINUX-STEP-03A
    question: Can task input, script path, or referenced file be modified?
    next: LINUX-ROOT-CANDIDATE
  - id: LINUX-STEP-04
    question: Do local configuration files contain credentials or secrets?
    commands:
    - grep -RinE '(password|passwd|secret|token|key)' /var/www /opt /home 2>/dev/null
    - find / -name '.env' -o -name 'config*' 2>/dev/null
    if_yes: LINUX-STEP-04A
    if_no: LINUX-STEP-05
  - id: LINUX-STEP-04A
    question: Can recovered secrets authenticate to a more privileged account or protected
      service?
    next: LINUX-ROOT-CANDIDATE
  - id: LINUX-STEP-05
    question: Are dangerous Linux capabilities set on binaries?
    commands:
    - getcap -r / 2>/dev/null
    if_yes: LINUX-STEP-05A
    if_no: LINUX-STEP-06
  - id: LINUX-STEP-05A
    question: Does capability set permit privilege or file-access escalation?
    look_for:
    - cap_setuid
    - cap_dac_override
    - cap_sys_admin
    next: LINUX-ROOT-CANDIDATE
  - id: LINUX-STEP-06
    question: Is the user in a powerful group or attached to risky local services?
    commands:
    - id
    - groups
    - ps aux
    look_for:
    - docker
    - lxd
    - adm
    - disk
    if_yes: LINUX-ROOT-CANDIDATE
    if_no: LINUX-STEP-07
  - id: LINUX-STEP-07
    question: Are there writable services, startup scripts, or unusual PATH/script
      behaviors?
    commands:
    - systemctl list-units --type=service
    - find / -writable -type f 2>/dev/null
    if_yes: LINUX-ROOT-CANDIDATE
    if_no: LINUX-STEP-08
  - id: LINUX-STEP-08
    question: As a last resort, is the kernel/service version old enough to justify
      careful local exploit research in lab conditions?
    commands:
    - uname -a
    - cat /etc/os-release
    if_yes: LINUX-LAST-RESORT
    if_no: LINUX-END
  terminal_nodes:
  - id: LINUX-ROOT-CANDIDATE
    meaning: High-value escalation path likely exists; switch from broad enumeration
      to focused validation.
  - id: LINUX-LAST-RESORT

    look_for:

    if_yes: WIN-PRIV-CANDIDATE
    if_no: WIN-STEP-04
  - id: WIN-STEP-04

      rights?
    look_for:
    - SeImpersonatePrivilege
    - SeAssignPrimaryTokenPrivilege

    if_yes: WIN-PRIV-CANDIDATE

    question: Are scheduled tasks or startup entries writable or weakly protected?

    commands:

    if_yes: WIN-LAST-RESORT

  - Match detected services against service_fingerprint_map
  - Sort by priority descending
  - Launch recommended_enumeration modules for top matches

  - For each matched service, evaluate corresponding foothold_patterns
  - Score matches based on observed signals
  - Pursue top 2-3 highest confidence paths first
  phase_4_post_foothold:
  - Determine OS
  - Run appropriate privesc decision tree
  - Document evidence, credentials, and lateral reuse opportunities

  - If no foothold found, revisit hidden ports, exposed backups, credential reuse,
